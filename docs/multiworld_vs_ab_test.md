"""
多平行世界框架 vs 传统A/B实验：意义与分析
================================================

本文档阐述本项目中"多平行世界仿真框架"的设计初衷、
相对于传统A/B实验的优势，以及"核参数在线更新"的
因果推断意义。


## 一、传统A/B实验的隐含假设与局限

### 1.1 静态参数假设

传统A/B实验（及大多数因果推断方法）隐含假设：

    Y_i(a) = f(X_i, θ_i)    其中 θ_i 不随实验进行而改变

即：用户 i 的"真实参数" θ_i 在实验期间是**时不变**的。

这个假设在实践中经常被违反：

| 动态效应 | 机制 | 对A/B的偏差 |
|-----------|------|----------------|
| 疲劳累积 | 多轮补贴 → 用户对补贴脱敏 | 后期ATE被低估（attenuation bias） |
| 参考点适应 | 持续补贴 → 参考点上移 → 同等补贴主观价值↓ | ATE随时间递减，平均效应误导 |
| 心理账户迁移 | Windfall Spender → Routine Income | 策略对"冲动型用户"的效应被稀释 |
| 选择性退出 | 高疲劳用户离开平台 | 幸存者偏差，A/B只观测留存用户 |

### 1.2 SUTVA违反

A/B实验要求**SUTVA**（Stable Unit Treatment Value Assumption）：
"一个用户的潜在结果，不依赖于其他用户的干预状态。"

在补贴场景中，SUTVA经常被违反：
- **社会传染**：邻居领券 → 我也想领（network effect）
- **预算挤出**：平台总补贴预算固定 → 给A发券 → B拿到的券面额变小
- **市场扰动**：大规模发券 → 商家涨价 → 所有人的补贴效应被稀释

传统A/B实验无法量化SUTVA违反的程度。


## 二、多平行世界框架的设计初衷

### 2.1 初心：解耦"假设风险"与"随机噪声"

项目初期的核心洞察：

> "我想研究的不是'随机噪声让结果怎样'，
>  而是'如果假设不同，结果会怎样'。"

具体说：
- **随机噪声**：换个随机种子，ROI从2.1变成1.9 → 这是噪声
- **假设风险**：如果用户的真实疲劳率是0.15而不是0.05 → ROI会变成多少？

多平行世界的设计：
  世界A：θ_i = 基础值（假设用户不被策略改变）
  世界B：θ_i(t+1) = f(θ_i(t), 补贴历史)（参数被策略塑造）
  世界C：...（其他假设）

同一组Agent，相同随机种子，不同假设 → 差异纯粹来自"假设"。

### 2.2 初衷的延伸：参数被策略反作用

更深层的设计动机：

  补贴策略 → 改变用户状态 → 改变用户参数 → 改变后续响应

这是一个**因果动力学**问题，而不是静态因果问题：

  静态因果：E[Y_i(1) - Y_i(0)] = ?
  动态因果：dθ_i(t)/dt = g(补贴历史) → θ_i(t) 影响 Y_i(t+1)

传统A/B只回答静态问题。
多平行世界通过"核参数在线更新"，可以回答动态问题。


## 三、核在线更新的因果意义

### 3.1 算法回顾

`KernelOnlineUpdater` 在每轮仿真结束后，根据Agent行为序列更新其UserKernel：

   new_theta_{i,s} = (1-lr) * old_theta_{i,s} + lr * observed_rate_s
   new_beta_{i,s}  = (1-lr) * old_beta_{i,s}  + lr * observed_uplift_s

这本质上是**指数加权移动平均（EWMA）**，
是共轭先验下贝叶斯在线更新的近似。

### 3.2 为什么这是因果的？

关键：多平行世界提供了**反事实对比**。

单一世界中跑在线更新，只能看到"策略A下的参数演化"，
无法区分"演化是策略导致的"还是"时间趋势"。

但在多平行世界中：

   世界A（高补贴策略）:  theta_clicked_i(t) 的轨迹
   世界B（低补贴策略）:  theta_clicked_i(t) 的轨迹
   世界C（无补贴对照）: theta_clicked_i(t) 的轨迹

如果：
   theta_i^A(t) 随 t 递减（补贴效应衰减）
   theta_i^C(t) 基本稳定
   → 这提供了"补贴导致敏感度下降"的因果证据

因为三个世界的**唯一区别是策略**，Agent的初始状态完全相同。

### 3.3 传统A/B实验为何做不到

传统A/B实验的通常做法：

   随机分桶 → 策略A/B分别投放 → 比较桶间平均效应

存在的问题：
1. **只测量静态效应**：E[Y|A] - E[Y|B]，一个数字
2. **无法观测参数演化**：用户的θ_i(t)是黑盒
3. **SUTVA违反无法量化**：网络效应混杂在ATE里
4. **动态异质性被忽略**：新用户 vs 老用户，对补贴的适应速度不同


## 四、多平行世界能回答的传统A/B不能回答的问题

### 4.1 问题清单

| # | 问题 | 传统A/B | 多平行世界 |
|---|------|----------|------------|
| 1 | 策略A和B的效应，在不同用户生命周期阶段是否不同？ | ❌ 只能看平均 | ✅ 看theta_i(t)轨迹的分叉点 |
| 2 | 补贴策略是否导致用户"耐药性"产生？ | ❌ 后期ATE下降，但归因不清 | ✅ beta_i(t)递减 + 跨世界对比 |
| 3 | 不同策略对用户心理账户迁移的不同影响？ | ❌ 心理账户不可观测 | ✅ gamma_i(t)轨迹 + 账户类型迁移统计 |
| 4 | 哪种策略最"温和"，不会导致用户疲劳累积？ | ❌ 只能看最终留存 | ✅ fatigue(t)轨迹对比 |
| 5 | 社会网络效应（SUTVA违反）有多大？ | ❌ 无法分离网络效应 | ✅ 世界A（有网络）vs 世界B（无网络） |
| 6 | 如果策略A运行30轮后再切换到策略B，效应如何？ | ❌ 实际切换成本高，不会做 | ✅ 世界A运行30轮 → 取最终核 → 在新世界中继续运行 |

### 4.2 最关键的第6个问题

这是多平行世界框架**最独特**的能力：

   传统A/B：策略A vs 策略B，同时启动，比较同期效应
   多平行世界：策略A运行30轮 → 观测"被A塑造后的用户"
                 → 在这些用户上运行策略B → 观测"切换效应"

这意味着可以回答：

   "如果一个用户已经被高补贴策略A'训练'了30轮，
    再切换到低补贴策略B，他的响应会怎样？"

这是**策略切换成本**问题，对实际业务极其重要，
但传统A/B实验几乎无法研究（切换成本高，且违反伦理）。

## 六、总结

多平行世界框架的核心价值：

  传统A/B实验：回答"策略A比B好多少"（静态平均）
  多平行世界：  回答"策略如何改变用户"（动态参数演化）

这不是"更精确的A/B"，而是**完全不同的一类因果问题**——
从静态因果推断，走向**因果动力学**。

在补贴策略场景中，这种动态视角是至关重要的：
因为补贴的目的，本来就是"改变用户行为"——
如果这个改变本身会改变用户对后续补贴的响应，
那么忽略这种动态，就会导致策略评估的根本性偏误。
